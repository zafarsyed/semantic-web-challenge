  • Competition guidelines

    1. Each team must have at least 2 and maximum of 4 participants
    2. A participant can register only in one team
    3. Participants are free to perform any preprocessing of the data provided as they see fit
    4. Only the training data provided by the organizers must be used for training the models
    5. Participants are strongly encouraged to use publicly available tools which can be shared with the community

  • Submission

    When you registered, you will get remote access to a virtual machine (Windows or Linux) to deploy the task software in. Your software must be executable from the command line and not require Internet access during the evaluation period.

    Note that all models must be capable of CPU inference only as we do not support GPU based models.

  • Evaluation using TIRA

    You will be able to self-evaluate your software using the TIRA service. You can find the user guide here. The automatic evaluation will use ROUGE measure and a script will be provided shortly after the start of the competition. Participants will be able to perform self evaluation using this script and report the F-1 scores accordingly for ROUGE-1, ROUGE-2 and ROUGE-LCS.

  • Competition timeline

    November 5th, 2018 : Training data available, competition begins
    January 15th, 2019 : Submission system and public leaderboard open
    April 1st, 2019 : Submission system closed (23:59 PM UTC), manual evaluation begins
